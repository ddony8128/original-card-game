# 밸런스 분석 리포트 (2026-06)

> 시뮬레이션 기반 분석. **데이터/카드 수치는 적용하지 않았음** — 적용은 설계 판단 + DB 동기화(마이그레이션)가 필요하므로 별도 결정 후 진행.
> 분석 도구: `server/src/core/ai/selfPlay.ts`(`playOneGameTraced`/`runSelfPlay`), 헤드리스 vitest. 라이브 서버(3000) 미사용.

## 재현 방법

- TS 스크립트 직접 실행 수단(tsx/ts-node)이 없어 `selfPlay.test.ts` 와 동일한 mock 패턴(`cardsService.listAll` → `cards.json`)으로 vitest 스펙을 작성해 구동.
- 핵심: `playOneGameTraced(a, b, getCardMeta, seed)` 를 매치업·시드별 반복. 기본덱 상대 프로필 `basic`. 결정적 PRNG(`mulberry32`)라 시드 동일 시 100% 재현.
- 판수: 매치업당 100판 × 시드 2세트(50000/200000). 두 시드 결과 ±10%p 이내 일관.
- 상수: `MAX_TURNS=80`(초과 시 무승부).

## A. 매치업 승률 / 평균 턴수

매치업당 100판. "A승" = 표 왼쪽(스테이지 AI) 승률.

| 매치업 | A승률(seed50k/200k) | 무 | 평균 턴 |
|---|---|---|---|
| 1 무투법사 vs 기본덱 | **24% / 33%** | 0 | ~8.7 |
| 2 게임개같이하네 vs 기본덱 | 52% / 39% | 7 | ~27 |
| 3 노루약해요 vs 기본덱 | **86% / 86%** | 0 | ~15 |
| 1 vs 2 | 42% / 48% | 0 | ~13 |
| 1 vs 3 | 19% / 22% | 0 | ~11 |
| 2 vs 3 | 8% / 11% | **무 60~66** | ~62 |

게임 길이(900판 합산): 원턴킬(≤2턴) 0%, 3~10턴 ~36%, 11~40턴 ~52%, 41~79턴 ~3%, **80턴 cap 무승부 ~8.6%**.

### 해석 (이상치)
1. **난이도 곡선 양극화.** "1단계" 무투법사가 기본덱 상대 24~33%로 제일 약하고, "3단계" 노루약해요가 86%로 압도. 1단계=거의 무료 승리, 3단계=거의 무료 패배 수준.
2. **2 vs 3 교착(무 60%+).** 둘 다 컨트롤/카이팅(밀·힐·드로우)이라 80턴 cap까지 결판 안 남. 피니셔 부족.
3. **어그로(무투)가 컨트롤(노루)에 구조적 열세**(1 vs 3 ≈ 20%). 노루의 힐(운기조식·지력흡수) + 운석 피니셔가 어그로 클럭을 압도.

### 하네스 한계 (중요)
- **하드 스테이지(4/5/6)의 `aiHp=30` 이 selfPlay 에 미반영.** `aiHp` 는 `soloGameManager.ts` 에만 적용되고 `selfPlay` 의 초기 상태 생성엔 안 들어가, 4/5/6 결과가 1/2/3 과 완전히 동일하게 나옴. 실제 게임은 AI HP +10 만큼 더 어려움. (→ 시뮬 충실도를 높이려면 selfPlay 초기상태에 aiHp 반영 필요)

## B. OP / 약체 카드 후보

trace 의 action 문자열에 카드 사용 정보가 있어 "그 카드를 쓴 쪽이 그 게임에서 이겼는지"를 집계(900판).

| 분류 | 카드 | 사용게임 | 사용시 승률 |
|---|---|---|---|
| OP | 운석 떨구기(c01-026) | 520 | **77.7%** |
| OP | 지력흡수(c01-015) | 642 | 62.0% |
| OP | 지진(c01-023) | 703 | 61.6% |
| 약체 | 각력 강화(c01-004) | 302 | **22.2%** |
| 약체 | 고통 동기화(c01-011) | 304 | **25.0%** |
| 약체 | 마나 담긴 찌르기(c01-002) | 248 | 25.0% |
| 약체 | 카드 날리기(c01-012) | 308 | 26.9% |
| 약체 | 전광석화(c01-017) | 157 | 29.3% |

**데이터 한계(반드시 감안):** "사용시 승률"은 인과가 아니라 상관. 약체 후보 대부분이 약체 덱(무투)에 들어 있어 낮게 나옴 — 카드가 나쁜 게 아니라 그 덱이 지는 것일 수 있음. 카드 단독 파워 분리는 "동일 덱에 카드만 스왑" A/B 시뮬 필요. AI 는 휴리스틱이라 사람 플레이와 차이(특히 무승부 교착은 AI 피니시 탐색 한계일 수 있음).

## C. 카드 설명 ↔ 동작 불일치

cards.json `effect_json`(실제 동작) vs `description_ko/en` 대조. **확정된 불일치만:**

| 카드 | 설명 | 실제 동작 | 판단 |
|---|---|---|---|
| **c01-028 엔드게임** | "거리3 내 **4** 피해 … 두 번 더 반복" (=4+4+4=12) | damage **4 → 5 → 5** (합 14) | 불일치. 설명을 4/5/5로 고치거나, 데이터를 4/4/4로 맞추거나 — **의도 확인 필요** |

> 참고: c01-006 마력탄은 ko "피해 4" = effect 4 로 **일치**(초기 의심은 오독). 최근 커밋(고통동기화/지맥/쓰레기버려욧 정정)들도 현재 cards.json/테스트와 일관.

## D. 조정 제안 (cards.json 필드 기준 — 적용 안 함)

> 모두 **단일 변수만 바꾸고 위 vitest 하네스로 재측정** 권장. 그리고 **cards.json 수정은 DB(원본) 마이그레이션이 동반돼야 라이브에 반영됨.**

1. **1단계(무투) 강화 — 가장 시급**
   - (덱) stage-1 의 고통동기화(c01-011) 비중 축소(자해 5로 어그로 자기클럭 손해, 25% 승률) 또는
   - (카드) c01-011 자해 5 → 3, c01-002 마나담긴찌르기 2 → 3.
2. **3단계(노루) 약화 — 86%는 과함**
   - (카드) c01-026 운석떨구기 value 10 → 8 또는 range 4 → 3 (단일 최강 피니셔), c01-015 지력흡수 heal 3 → 2 (무한 힐 엔진·교착 원인).
   - (덱) stage-3 의 운석/좋은게임 비중 축소.
3. **2 vs 3 교착 완화**
   - 두 컨트롤 덱에 확정 피니셔 부족. stage-2 에 운석 1장 추가 등. (cards.json 만으론 피니셔 추가가 답; 또는 80턴 cap 시 HP 적은 쪽 패배 규칙 — 엔진 차원)
4. **약체 어그로 보정(선택)**
   - c01-017 전광석화 1/2/3 → 2/3/4, c01-012 카드날리기 discard 2 → 1.
