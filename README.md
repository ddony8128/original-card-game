이 프로젝트는 `client/`와 `server/`로 분리된 구조의 카드 게임 기본 세팅입니다.

- client: React 19 + TypeScript 5.9, Vite 7 기반. 상태관리는 Zustand, 라우팅은 React Router, HTTP 통신은 Axios를 사용합니다. ESLint 9 설정과 `dev/build/preview` 스크립트가 포함되어 빠른 개발·빌드를 지원합니다.
- server: Express 5와 CORS를 사용하는 최소 서버로, 추후 게임 API·세션 관리 등으로 확장 가능합니다.

역할 분리: 클라이언트는 UI/게임 로직을 담당하고, 서버는 백엔드 엔드포인트와 게임 상태 관리를 담당합니다. 폴더 구조는 간단하며, `client/`에는 프론트엔드 소스와 Vite/TS 설정이, `server/`에는 Express 의존성과 서버 코드가 위치합니다.

## 클라이언트 화면 개요

- /lobby: 닉네임 등록, 덱 만들기 이동, 방 만들기/참가. 방 생성 시 코드가 발급되며, 덱이 있어야 생성/참가 가능. 성공 시 `/game`으로 이동.
- /deck-list: 저장된 덱 목록 조회·삭제·수정(최대 4개). 새 덱 만들기(`deck-builder`) 및 로비로 돌아가기 버튼 제공.
- /deck-builder: 카드 선택으로 덱 구성. 제약: 총 16장, 동일 카드는 최대 2장. 새 덱 생성 또는 기존 덱 수정 가능. 저장 시 `/deck-list`로 이동.
- /game: 임시 게임 화면. 현재 입장한 방 코드 표시, 추후 보드/마나/HP UI 추가 예정. 임시로 결과 화면으로 이동 버튼 제공.
- /result: 임시 결과 화면. 승패/전적 통계 표시 예정, 로비로 돌아가기 버튼 제공.

상태 관리: 덱은 `useDeckStore`(persist로 로컬 저장), 사용자/방 정보는 `useGameStore`에서 관리합니다.
